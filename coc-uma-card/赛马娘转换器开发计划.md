# COC 角色卡转赛马娘育成卡开发计划

## 成品目标

最终卡面按新版效果图推进：

- 顶部保留 `COC角色详情` 标题栏。
- 左侧保留赛马娘头像和评价点。
- 中上区域放总评级大徽章，例如 `UG RANK`。
- 右侧展示 COC 角色名。
- 职业放在橙色横幅中。
- 五维栏保持赛马娘属性：`スピード / スタミナ / パワー / 根性 / 賢さ`。
- 每个五维格展示一个小评级徽章和一个数值。
- 底部适性区改成多个小卡片，每张卡片显示项目名和等级。

## 卡面数据结构

### 顶部信息

| 区域 | 内容 | 来源 |
| --- | --- | --- |
| 总评级徽章 | `G~UG RANK` | 评价点换算 |
| 角色名 | COC 角色名 | Excel 的姓名字段 |
| 玩家名 | `玩家：xxx` | Excel 的玩家字段 |
| 职业 | COC 职业 | Excel 的职业字段 |
| 评价点 | `00,000` 格式 | 综合换算 |

### 五维信息

五维使用赛马娘模板属性，数值由 COC 属性换算：

| 赛马娘属性 | 日文（模板上显示） | 中文 | COC 来源 | 展示内容 |
| --- | --- | --- | --- | --- |
| 速度 | スピード | 速度 | DEX（敏捷）为主 + STR | 小评级徽章 + 换算数值 |
| 耐力 | スタミナ | 耐力 | CON（体质）为主 + STR + SIZ | 小评级徽章 + 换算数值 |
| 力量 | パワー | 力量 | STR（力量）为主 + CON | 小评级徽章 + 换算数值 |
| 根性 | 根性 | 根性/毅力 | POW（意志）为主 + CON | 小评级徽章 + 换算数值 |
| 智力 | 賢さ | 贤明/智慧 | INT + EDU + APP 均分 | 小评级徽章 + 换算数值 |

五维评级规则：

- `1100+`：SS
- `900~1099`：S
- `750~899`：A
- `600~749`：B
- `450~599`：C
- `300~449`：D
- `<300`：E

### 适性信息

底部按 COC 实际玩法拆成 10 个适性项目：

| 分类 | 项目 | 主要参考 |
| --- | --- | --- |
| 战斗适性 | 近战 | 斗殴, 近战武器, STR, DEX |
| 战斗适性 | 远程 | 射击, 投掷, DEX |
| 探索适性 | 城市探索 | 侦查, 聆听, 心理学, 话术 |
| 探索适性 | 野外探索 | 追踪, 导航, 生存, 自然学 |
| 探索适性 | 文献调查 | 图书馆使用, 历史, 神秘学, EDU |
| 探索适性 | 资料收集 | 计算机, 会计, 法律, 语言类技能 |
| 生存适性 | 耐久生存 | CON, 急救, 医学 |
| 生存适性 | 应急处理 | 急救, 机械维修, 电气维修 |
| 生存适性 | 精神抗性 | POW, SAN, 克苏鲁神话 |
| 生存适性 | 运气 | 幸运, POW |

适性等级使用 `S/A/B/C/D/E/F/G`。

## 开发 Plan

### Phase 1：重做新版模板

目标：把模板改成最终成品结构。

- 以新版效果图为目标，生成新版空白底图。
- 清掉示例文字和示例数字。
- 保留头像, 背景, 标题栏, UI 框架。
- 标定角色名, 职业, 评价点, 总评级, 五维, 适性卡片坐标。
- 输出新版预览图和坐标表。

验收标准：

- 图片尺寸固定为 `1199 x 1312`。
- 总评级大徽章位置稳定。
- 五维四位数不会压线。
- 五维标题保持 `スピード / スタミナ / パワー / 根性 / 賢さ`。
- 10 个适性卡片文字完整。

### Phase 2：生成评级徽章素材

目标：让卡面有接近游戏 UI 的完成度。

- 生成总评级大徽章：`G / F / E / D / C / B / A / S / SS / UG`。
- 生成五维小评级徽章：`E / D / C / B / A / S / SS`。
- 统一描边, 渐变, 阴影, 高光。
- 徽章素材全部用透明 PNG。

验收标准：

- 大徽章能放进顶部评级区。
- 小徽章能放进五维格。
- 深色背景和浅色背景上都能看清。

### Phase 3：COC Excel 解析

目标：从角色卡拿到完整字段。

- 复用 FUT 版本里的 Excel 解析逻辑。
- 新增玩家名提取，先保留到数据结构里。
- 读取姓名, 职业, 基础属性, 技能, SAN, 幸运。
- 给缺失字段设置可读默认值。

验收标准：

- 至少 3 份不同格式 COC 卡都能解析。
- 解析失败时返回明确错误。
- 基础属性和技能能分开读取。

### Phase 4：换算规则

目标：把 COC 数据变成卡面评级。

- 根据 COC 属性换算赛马娘五维。
- 根据赛马娘五维计算小评级。
- 根据基础属性, 技能, SAN, 幸运计算总评价点。
- 根据总评价点计算总评级。
- 根据技能组计算 10 个适性等级。

总评级建议：

| 总评级 | 评价点 |
| --- | --- |
| G | 0+ |
| F | 3,000+ |
| E | 5,000+ |
| D | 7,000+ |
| C | 10,000+ |
| B | 12,500+ |
| A | 14,500+ |
| S | 17,500+ |
| SS | 19,600+ |
| UG | 23,900+ |

验收标准：

- 同一角色重复转换结果稳定。
- 低数值角色不会被抬得过高。
- 高技能角色能体现优势。

### Phase 5：图片渲染

目标：输出最终 PNG 成品。

- 编写 `render_uma_card()`。
- 读取新版空白模板。
- 贴总评级大徽章。
- 贴五维小评级徽章。
- 写入角色名, 职业, 评价点, 五维数值, 适性项目等级。
- 长文本自动缩小字号。

验收标准：

- 生成图片清晰。
- 所有文字在框内。
- 长角色名不会溢出。
- 图片能直接发到群里。

### Phase 6：Flask API

目标：把图片生成包装成接口。

- 新增 `/generate_uma`。
- 接收 Excel 文件。
- 返回 PNG 文件或图片 URL。
- 失败时返回简短错误信息。

验收标准：

- 上传 Excel 后能返回完整 PNG。
- 服务端日志能定位解析失败和渲染失败。

### Phase 7：海豹骰插件联调

目标：让群里能直接用 `.赛马娘`。

- 编写 `.赛马娘` 指令。
- 接收群文件。
- 调用 Flask API。
- 返回生成图片。
- 增加超时和失败提示。

验收标准：

- 群内上传 COC 卡后能生成赛马娘卡。
- API 挂掉时给出简短提示。
- Excel 格式不对时提示用户换文件。

## 第一优先级

下一步先做 Phase 1 和 Phase 2。

原因：

- 新版模板结构已经确定。
- 坐标和素材先固定，后端渲染才不会返工。
- 徽章是最终质感的关键部分。
